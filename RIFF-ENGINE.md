## The Riff

A riff is a constructive contribution that builds on what came before. It's not a response — it's an evolution. Every riff takes the current state and advances it.

Two modes of riffing:
1. **Creative**: song, play, D&D, story — agents build on each other's artistic contributions
2. **Technical**: decompose an application into the cellular graph — agents break down and build up

Both modes use the SAME underlying mechanism:
- Agent A riffs → produces an artifact (text, code, plan)
- The artifact enters the cellular graph as a "tick"
- The graph routes it to the right agents (murmur)
- Agent B receives A's riff as context, adds their own contribution
- JEPA predicts what would make the riff better → nudges agents toward productive directions
- Over iterations, the collaboration improves because the graph learns which agent combinations produce the best riffs

## The Riff as a Room

Each riff session is a room in the cellular graph:
- **Perception DB**: all contributions so far (the growing artifact)
- **Prediction DB**: what each agent expects the next contribution to be
- **JEPA**: measures surprise — when an agent's contribution is unexpected, the riff takes a creative turn
- **Vibe**: the emotional/technical character of the session (playful, intense, analytical, exploratory)
- **GC**: when the session gets too long, merge similar ideas, prune dead ends

## Iteration Growth

The key insight: the system IMPROVES across sessions because:
1. After each riff session, the cellular graph learns which agent pairings work
2. The JEPA's prediction accuracy increases — it gets better at predicting productive directions
3. The spreader-tool distributes successful patterns to new sessions
4. LoRA fine-tuning crystallizes the best collaboration strategies
